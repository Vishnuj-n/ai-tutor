# Code Quality Audit: "God Files" and Monolithic Bloat

The following files have been identified as "God Files" based on the specified criteria (Size > 500 LOC for Go / 300 LOC for Vue, Duty Overload, Exported Function Density, and Repeated Logic Patterns). `internal/db/store.go` has been excluded as requested.

| File Path | Primary Duty | Secondary/Hidden Duties | Refactor Recommendation |
| --- | --- | --- | --- |
| `app.go` (2297 LOC, 19+ Exported funcs) | Main Wails application lifecycle and system binding. | LLM prompt generation, orchestrating RAG pipelines, scheduling wrappers, and repeated JSON error wrapping boilerplate (`map[string]interface{}`). | Split Wails lifecycle from business controllers. Move prompt generation to a dedicated LLM prompt service and create specific vertical handlers. |
| `notebook_endpoints.go` (894 LOC) | Wails backend API endpoints for notebook management. | File ingestion event broadcasting, topic extraction logic, OS-level command execution (`os/exec`), and deep JSON boilerplate for error handling. | Extract file ingestion background orchestration to a dedicated service. Use standardized error types instead of repetitive map wrapping. |
| `internal/notebook/upload.go` (655 LOC) | Managing file uploads and creating notebook records. | Zip file extraction, PDF text parsing, manual chunking/splitting of text strings, and metadata extraction. | Separate file handling (multipart/saving) from document parsing. Move PDF parsing and chunking into a dedicated `parser` package. |
| `internal/embeddings/onnx.go` (641 LOC) | Loading and executing ONNX models for embeddings. | Complex Tokenizer implementation (BPE), CGO memory management, and session pooling/locking logic. | Extract tokenization logic into its own package/file. Isolate memory management and CGO handling into an adapter layer. |
| `frontend/src/pages/Notebook.vue` (1212 LOC) | UI for the Notebooks management page. | State management for multiple sub-views, complex drag-and-drop logic, API retry handling, and ingestion progress polling. | Break down into smaller components (`TopicTree`, `NotebookUpload`). Move complex state into a Pinia store. |
| `frontend/src/pages/Reader.vue` (825 LOC) | Document reading and viewer interface. | Paging state logic, semantic highlighting calculations, inline flashcard/quiz triggering, and direct API communication. | Extract core viewer into a `DocumentViewer` component. Move API calls and complex state to a composable or store. |
| `frontend/src/pages/Socratic.vue` (628 LOC) | UI for the Socratic AI chat. | Message history management, auto-scrolling DOM manipulations, LLM streaming response parsing, and connection state. | Split into `MessageList` and `MessageInput`. Extract streaming text and chat state logic into a dedicated store. |
| `frontend/src/pages/Dashboard.vue` (535 LOC) | Main dashboard entry point and overview. | Aggregating stats across disparate domains (notebooks, flashcards, assessments) and complex responsive layout handling. | Componentize into smaller domain widgets (`RecentActivity`, `StudyStatsWidget`) and fetch data at the widget level. |
| `frontend/src/pages/Flashcards.vue` (524 LOC) | Flashcard review interface. | FSRS scheduling state tracking, card flip animation logic, and review session progression state. | Extract session progress to a store. Create a purely presentational `Flashcard` component for animations. |
| `frontend/src/pages/WrittenAssessment.vue` (491 LOC) | UI for taking written assessments. | Timer management, answer grading visualization (diffing), and API payload formatting. | Extract timer to a reusable composable. Break out the grading feedback into a separate `GradingFeedback` component. |
| `frontend/src/pages/Quiz.vue` (424 LOC) | Multiple-choice quiz interface. | Quiz generation configuration, active taking state, and scoring logic calculation. | Separate the quiz creation/configuration view from the active test-taking view. Extract scoring into a utility module. |
